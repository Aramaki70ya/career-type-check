## 補足：質問設計の意図

各質問は、対応するタイプの以下の特徴を測定するよう設計しています：

| タイプ | 測定している特徴 |
|--------|------------------|
| パイオニア | 未知への挑戦意欲、開拓精神、リスクテイク |
| ドライバー | 目標志向、成果主義、競争心 |
| インフルエンサー | 発信力、プレゼン力、巻き込み力 |
| コネクター | 調整力、根回し力、関係構築力 |
| サポーター | 奉仕精神、裏方志向、気配り |
| エデュケーター | 育成意欲、指導力、説明力 |
| ヴィジョナリー | 戦略思考、構造化力、ビジョン構築 |
| クリエイター | 独創性、自由志向、創作意欲 |
| ハッカー | 改善志向、分析力、効率化 |
| スペシャリスト | 専門性追求、品質へのこだわり |
| ガーディアン | 規律遵守、リスク察知、正確性 |
| アーキテクト | 仕組み化、論理性、再現性重視 |

## レビュー観点（参考）

質問文をレビューする際の観点：

1. **タイプとの整合性**: その質問が本当に該当タイプの特徴を測定しているか
2. **重複の排除**: 異なるタイプ間で似たような質問になっていないか
3. **中立性**: 特定の回答に誘導するような表現になっていないか
4. **理解しやすさ**: 曖昧な表現や専門用語が含まれていないか
5. **社会的望ましさバイアス**: 「正解」が透けて見えるような質問になっていないか
